Built by Al Fayaz, NEXIV Digital Agency

## How Mode Switching Works

NEXIV Claude Protocol uses a layered mode system. The core protocol is always the base layer. Domain modes add specialized rules on top of the core. You can only be in one domain mode at a time, but the core is always active when any mode is active.

Think of it as stacking:

Core layer (always active when protocol is on): compression rules, output structure, prompt amplification, decision acceleration, personalized context

Domain layer (one active at a time): specialized vocabulary, output formats, domain knowledge, context specific to that discipline

## Activation Triggers

### Activating the Core Protocol

Say any of the following:
- /nx
- nexiv mode
- al fayaz mode
- nexiv protocol
- activate nexiv
- less tokens
- be brief

### Activating Domain Modes

Say the command for the domain you are entering:

/nx pipe or pipeline mode or automation mode or n8n mode: Activates Pipeline Mode for n8n, Claude API, and automation work

/nx manga or manga mode or story mode or deus mortis mode: Activates Manga Mode for visual storytelling and Deus Mortis work

/nx agency or agency mode or business mode or client mode: Activates Agency Mode for NEXIV business and strategy work

/nx prompt or prompt mode or prompt engineering mode: Activates Prompt Mode for meta-prompt engineering work

/nx code or code mode: Activates Code Mode for code-heavy work sessions

/nx ultra or ultra mode: Activates maximum compression with no specific domain mode. Applies to all outputs.

### Switching Between Modes

You can switch modes mid-conversation. Say the new mode command and Claude switches immediately. The core protocol stays active. Only the domain layer changes.

Example:
- Start with /nx pipe to design a pipeline
- Switch to /nx manga to write a character that appears in a story referenced in pipeline data
- Switch back to /nx pipe to continue the architecture

### Deactivating the Protocol

Say any of the following to fully deactivate:
- stop nexiv
- normal mode
- stop caveman

Claude returns to standard behavior immediately and completely.

## When to Use Which Mode

Use /nx pipe when: designing n8n workflows, writing Claude API calls, debugging automation, architecting multi-agent systems, reviewing pipeline costs

Use /nx manga when: creating characters for Deus Mortis, writing scenes or panels, building world lore, generating image prompts for Nanno Banan Pro, developing story arcs

Use /nx agency when: making business decisions for NEXIV, developing GeoSignal strategy, writing client proposals, evaluating partnerships, operational planning

Use /nx prompt when: writing system prompts for Claude API, auditing existing prompts, designing multi-agent instruction systems, reviewing prompt costs and token efficiency

Use /nx code when: code-heavy sessions where you want to make the code boundary explicit, debugging code, reviewing code architecture

Use /nx ultra when: you want maximum compression with no specific domain focus, very high volume sessions where every token matters, quick reference questions
